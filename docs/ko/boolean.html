부울

부울은 bool이라는 타입을 가지며 true 혹은 false가 될 수 있습니다. 기본 연산은 다음이 있습니다:

  • &&: 논리곱
  • ||: 논리합
  • !: 부정
  • <=, >=, <, >
  • ==: 구조적 동치, 데이터 구조를 깊게 비교합니다: (1, 2) == (1, 2) 은 true입니다. 편리하지만, 조심할 필요가 있습니다.
  • ===: 레퍼런스 동치, 얕게 비교합니다. (1, 2) === (1, 2) 은 false입니다. let myTuple = (1, 2); myTuple === myTuple 은 true입니다.
  • !=: 구조적 동치 부정
  • !==: 레퍼런스 동치 부정

사용

주의: Reason/Bucklescript의 true와 false는 BuckleScript 3.0 이후부터 JS의 true와false로 바뀝니다. 해당 릴리즈 전이라면 JS.to_bool과 Js.Boolean.to_js_boolean을 써야 하지만, 다행스럽게도 이제는 그렇지 않습니다!

팁 & 트릭

구조적 동치를 적절히 사용하세요. 이는 편리하지만, 어떤 경우엔 뜻하지 않게 두 개의 깊은 데이터 구조를 비교하게 하여 큰 성능 하락을 초래하게 될 수 있습니다. 또한 어떤 것이 "동치"로 여겨지는지도 항상 자명하지 않습니다. 예를 들어, 하나의 데이터 foo는 평가가 지연된(lazy) foo와 같을까요? 이상적으로는 이것에 대한 확장이 가능해야 합니다. 이것이 가능하고 믿을 수 있게 하는 변화가 만들어지고 있습니다. 관심이 있으시다면, modular implicit를 확인해보세요.
