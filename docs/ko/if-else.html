If-Else

Reason의 if는 식입니다, if는 몸통의 요소에 의해 평가됩니다:

let message = if (isMorning) {
  "Good morning!"
} else {
  "Hello!"
};

주의 if-else 식이 마지막에 else 로 끝나지 않을 경우 암시적으로 ()를 반환합니다.

if (showMenu) {
  displayMenu()
};

는 기본적으로 다음과 같습니다.

if (showMenu) {
  displayMenu()
} else {
  ()
};

예상하신대로 displayMenu()는 unit을 반환합니다.

그걸 확인하는 다른 방법이 있습니다. 아래 코드는 확실히 잘못되었죠.

let result = if (showMenu) {
  1 + 2
};

이 식은 타입 오류를 냅니다. 암시적으로 존재하던 else 브랜치는 unit을 반환하므로 if 브랜치가 반환하는 int와 타입이 다르기 때문입니다. 이건 직관적으로 이해가 됩니다. 만약 showMenu가 false라면 result에는 대체 무슨 값이 들어가겠습니까?

또 Reason에는 삼항식이라는 문법적 설탕이 있습니다.

let message = isMorning ? "Good morning!" : "Hello!";

사용

다른 언어에 비해 Reason에서 if-else와 삼항식은 매우 적게쓰입니다. 왜냐하면 예전에 쓰던 조건문은 패턴 매칭으로 처리할 수 있을 거니까요. 단 두 가지의 경우만 있을 때만 if-else를 선택하세요.

디자인 결정

Reason의 삼항식은 그냥 bool variant와 switch의 문법적 설탕입니다.

switch (isMorning) {
| true => "Good morning!"
| false => "Hello!"
}

refmt를 돌리면, 아래 코드가 나올 겁니다.

isMorning ? "Good morning!" : "Hello!";

흥미롭나요? 여기, refmt에 관한 블로그 게시물이 있습니다.
