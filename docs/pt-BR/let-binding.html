Let Binding

A "let binding", in other languages, might be called a "variable declaration". let gives names to values. They can be seen and referenced by code that comes after them.

let greeting = "hello!";
let score = 10;
let newScore = 10 + score;

Escopo de Bloco

Bindings têm seu escopo delimitado por {}.

if (mostrarCumprimento) {
  let mensagem = "Gostando da documentação?";
  print_endline(message)
};
/* `mensagem` não acessível aqui! */

Bindings são Imutáveis

"Imutável" como em, "não pode ser alterado". Uma vez que um binding faz referência a um valor, este não pode referenciar qualquer outro valor (a não ser que este contenha explicitamente um valor mutável, discutido posteriormente). Entretanto, você pode criar uma nova definição com o mesmo nome que esconde (shadow) o binding anterior; desse ponto em diante, a definição irá se referir ao novo valor atribuído.

let mensagem = "olá";
print_endline(mensagem); /* Imprime "olá" */
let mensagem = "adeus";
print_endline(mensagem); /* Imprime "adeus" */

Dicas & Truques

Dado que definições tem escopo delimitado por {}, você pode criar um escopo anônimo ao redor delas:

let message = {
  let part1 = "hello";
  let part2 = "world";
  part1 ++ " " ++ part2
};
/* `part1` and `part2` não são acessíveis aqui! */

Isto previne algum uso errôneo destes bindings após as linhas acima.

Decisões de Design

Reason é basicamente OCaml por baixo do panos. Um let binding, na sintaxe de OCaml, parece com:

let a = 1 in
let b = 2 in
a + b

Isso deve ser conceitualmente lido como:

let a = 1 in
  let b = 2 in
    a + b

Que corresponde ao seguinte trecho em Reason:

let a = 1;
let b = 2;
a + b;

O que pode lembrá-lo de:

/* Reason syntax */
(a) =>
  (b) =>
    a + b;

Apesar de não serem estritamente a mesma coisa, esperamos que você veja que let é somente uma expressão e semelhante a uma função! Em Reason, nós alteramos in para ; por conta da familiaridade visual; mas não deixe de ver a elegância das expressões que está na base disso.
