List & Array

List

Listen sind:

  • homogen
  • unveränderlich (immutable)
  • schnell um Elemente vorne einzufügen
let myList = [1, 2, 3];

Reason-Listen sind einfach verkettete Listen.

Verwendung

Du würdest List für seine Größenanpassungsfähigkeit, sein schnelles Einfügen am Anfang, und seine schnelle Aufteilung verwenden, die alle unveränderlich und doch effizient sind!

Die Standardbibliothek stellt ein Listenmodul (und sein Gegenstück mit benannten Argumenten, nämlich ListLabels) bereit.

Unveränderliches Voranstellen

Verwende die Spread-Syntax, die lediglich List.cons entspricht:

let myList: list(int) = [1, 2, 3];
let anotherList = [0, ...myList];

myList hat sich nicht verändert. anotherList ist nun [0, 1, 2, 3]. Dies ist effizient (konstante Zeit, nicht lineare). Die letzten 3 Elemente in anotherList werden auch von myList verwendet!

Beachte dass[a, ...b, ...c] einen Syntaxfehler ergibt. Wir unterstützen kein mehrfaches Spread für eine Liste. Das wäre eine versehentliche lineare Operation (O(b)), denn jedes Element in b würde eins nach dem anderen dem Anfang von c hinzugefügt. Du kannst hierfür List.concat einsetzen.

Es wird davon abgeraten, ein beliebiges Element in der Mitte einer Liste zu ändern, denn dessen Performance und Speicheraufwand wäre linear (O(n)).

Zugriff

switch (beschrieben im Abschnitt zu Pattern-Matching) wird gewöhnlich verwendet, um auf Listenelemente zuzugreifen:

let message =
  switch (myList) {
  | [] => "This list is empty"
  | [a, ...rest] => "The head of the list is the string " ++ string_of_int(a)
  };

Um auf ein beliebiges Listenelement zuzugreifen, verwende List.nth.

Tips & Tricks

Du kannst ruhig so viele leere Listen erstellen wie du möchtest. Wie im Abschnitt zu Listen in Varianten erklärt, ist eine leere Liste tatsächlich ein parameterloser Variantenkonstruktor unter der Haube, der nur zu einer ganzen Zahl kompiliert wird. Keine zusätzliche Speicherzuteilung wird benötigt.

Um zu verstehen, wie das Voranstellen unveränderlich sein kann und O(1) zur gleichen Zeit, siehe auch den vorherigen Link.

Designentscheidungen

In Zukunft könnten wir eine vorgefertigte Listen-Datenstruktur bereitstellen, die unveränderlich, in der Größe veränderbar ist und rundherum schnelle Operationen hat.

Array

Arrays sind wie listen, außer dass sie

  • veränderlich sind
  • schnell in Bezug auf beliebigen Zugriff & Änderungen sind
  • eine feste Größe in nativer Umgebung haben (mit flexiblen Größen in JavaScript)

Du würdest sie mit [| und |] umgeben.

let myArray = [|"hello", "world", "how are you"|];

Verwendung

Die Module Array und ArrayLabel der Standardbibliothek. Für die JS-Kompilierung hast du auch die bekannte API Js.Array.

Zugriff & Änderungen mit Arrays geht so:

let myArray: array(string) = [|"hello", "world", "how are you"|];

let firstItem = myArray[0]; /* "hello" */

myArray[0] = "hey";

/* now [|"hey", "world", "how are you"|] */

Obiger Zugriff und Änderung sind bloß syntaktischer Zucker für Array.get bzw. Array.set.

Tips & Tricks

Wenn du nach JavaScript kompilierst, solltest du wissen, dass Reason-Arrays problemlos JavaScript-Arrays zugeordnet werden und umgekehrt. Auch wenn Arrays auf nativer Ebene eine feste Größe haben, kannst du die API Js.Array weiterhin verwenden, um ihre Größe zu ändern. Dies ist in Ordnung.
