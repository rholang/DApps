Newcomer Examples

Ein Beispiel ist besser als tausend Worte.

Dieser Abschnitt richtet sich an Neueinsteiger, die versuchen, allgemeine Idiome & Konventionen in Reason und BuckleScript zu verstehen. Wenn du ein Anfänger bist, der eine gute Idee für ein Beispiel hat, schlage bitte eine Änderung vor!

Den option-Typ verwenden

option ist eine Variante, die mit der Standartbibliothek kommt. Sie erübrigt die Notwendigkeit von Nullwerten in anderen Sprachen.

let possiblyNullValue1 = None;
let possiblyNullValue2: option(string) = Some("Hello@");

switch (possiblyNullValue2) {
| None => print_endline("Nothing to see here.")
| Some(message) => print_endline(message)
};

Einen parametrisierten Typen erstellen

type universityStudent = {gpa: float};

type response('studentType) = {
  status: int,
  student: 'studentType
};

let result: response(universityStudent) = fetchDataFromServer();

Ein JS-Objekt erstellen

Assuming you're compiling to JavaScript.

[@bs.deriving abstract]
type payload = {
  name: string,
  age: int
};

let obj1 = payload(~name="John", ~age=30);

/* Kompiliert zu einem JS-Objekt mit den obigen Feldern */

Beachte, dass obiges kein Record ist; es handelt sich um ein spezielles BuckleScript Feature.

Nullfähige JS-Typen mit dem option-Typ überprüfen

Für eine Funktion, deren Argument in JavaScript möglicherweise null oder undefined sein kann, ist es idiomatisch, sie als Reason-option zu konvertieren. Diese Konvertierung wird durch die Hilfsfunktionen in BuckleScripts Js.Nullable-Modul erreicht. In diesem Fall toOption:

let greetByName = (possiblyNullName) => {
  let optionName = Js.Nullable.toOption(possiblyNullName);
  switch (optionName) {
  | None => "Hi"
  | Some(name) => "Hello " ++ name
  }
};

Diese Überprüfung kompiliert zu possiblyNullName == null in JS und überprüft somit das Vorhandensein von null oder undefined.
