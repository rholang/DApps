Object

Die meiste Zeit in Reason verwendest du Record, um benannte Werte zusammenzufassen. In bestimmten Situationen möchtest du jedoch möglicherweise eine ähnliche Funktion namens object verwenden. Sie sind ein bisschen flexibler und bringen verschiedene Kompromisse mit sich.

Wenn du von JavaScript kommst, dann bist du höchstwahrscheinlich nicht auf der Ausschau nach einfachen Reason-Objekten. Gehe direkt zum Abschnitt Tips & Tricks unten.

Verwendung

Typdeklaration

Ein Objekt benötigt keine Typdeklaration, obwohl es eine haben kann. Es sieht wie ein Record aus, außer mit einem .:

type tesla = {
  .
  color: string
};

Der Punkt am Anfang deutet an, dass dies ein „geschlossener“ Objekttyp ist, was bedeutet, dass ein Objekt, das auf diesem Typ basiert, genau diese Form haben muss.

type car('a) = {
  ..
  color: string
} as 'a;

Zwei Punkte, auch Elision genannt, weisen darauf hin, dass es sich um einen „offenen“ Objekttyp handelt und daher auch andere Werte und Methoden enthalten kann. Ein offenes Objekt ist zusätzlich polymorph und benötigt daher einen Parameter.

Erzeugung

Einfach

type tesla = {
  .
  color: string,
};

let obj: tesla = {
  val red = "Red";
  pub color = red;
};

Js.log(obj#color) /* "Red" */

Hier haben wir ein einfaches Objekt mit der Methode color und der Eigenschaft red. Diese Methode erwartet kein Argument und gibt die private Eigenschaft red zurück. Weil die Methode color eine öffentliche Methode ist, können wir auf sie durch Objektnotation zugreifen. Zur Erinnerung: Objekte können nur Methoden exportieren und alle Eigenschaften sind privat.

Fortgeschritten

type tesla = {.
  drive: int => int
};

let obj: tesla = {
  val hasEnvy = ref(false);
  pub drive = (speed) => {
    this#enableEnvy(true);
    speed
  };
  pri enableEnvy = (envy) => hasEnvy := envy
};

Dieses Objekt ist vom Objekttyp tesla und hat eine öffentliche Methode drive. Es enthält außerdem eine private Methode enableEnvy, auf die nur innerhalb des Objekts zugegriffen werden kann.

Wie du sehen kannst, kann ein Reason-Objekt auch auf this zugreifen. In JavaScript-Objekten kann das Verhalten von this merkwürdig sein. In Reason zeigt this immer korrekt auf das Objekt selbst.

Das folgende Beispiel zeigt einen offenen Objekttyp, der einen Typ als Parameter verwendet. Der Objekttyp wird als Parameter benötigt, um alle Methoden des offenen Objekttyps zu implementieren.

type tesla('a) = {
  ..
  drive: int => int
} as 'a;

let obj: tesla({. drive: int => int, doYouWant: unit => bool}) = {
  val hasEnvy = ref(false);
  pub drive = (speed) => {
    this#enableEnvy(true);
    speed
  };
  pub doYouWant = () => hasEnvy^;
  pri enableEnvy = (envy) => hasEnvy := envy
};

Du kannst obiges Objekt so verwenden:

obj#doYouWant();

Tips & Tricks

Wenn du von JavaScript kommst, kompiliert ein Reason-Objekt nicht zu einem JS-Objekt. Probiere dafür BuckleScripts spezielles Feature.
