Let Binding

A "let binding", in other languages, might be called a "variable declaration". let gives names to values. They can be seen and referenced by code that comes after them.

let greeting = "hello!";
let score = 10;
let newScore = 10 + score;

Sichtbarkeitsbereich in einem Block

Für Bindungen kann mit {} ein Sichtbarkeitsbereich (scope) festgelegt werden.

if (displayGreeting) {
  let message = "Enjoying the docs so far?";
  print_endline(message)
};
/* `message` not accessible here! */

Bindungen sind unveränderlich

„Immutable“ im Sinne von „ändert sich nicht“. Sobald sich eine Bindung auf einen Wert bezieht, dann kann sie nicht auf etwas anderes verweisen (es sei denn sie enthält explizit einen veränderlichen Wert, was später beschrieben wird). Du kannst jedoch eine neue Bindung mit dem gleichen Namen erzeugen, die die vorige Bindung überschattet; Von diesem Zeitpunkt an bezieht sich die Bindung auf den neu zugewiesenen Wert.

let message = "hello";
print_endline(message); /* Prints "hello" */
let message = "bye";
print_endline(message); /* Prints "bye" */

Tips & Tricks

Da Bindungen durch {} definiert sind, kannst du einen anonymen Sichtbarkeitsbereich um sie herum erstellen:

let message = {
  let part1 = "hello";
  let part2 = "world";
  part1 ++ " " ++ part2
};
/* `part1` und `part2` sind hier nicht erreichbar */

Dies verhindert einen Missbrauch der Bindungen nach diesen Zeilen.

Designentscheidungen

Reason basiert unter der Haube auf OCaml. Eine Bindung mit let sieht in OCaml-Syntax folgendermaßen aus:

let a = 1 in
let b = 2 in
a + b

Konzeptionell könnte dies stattdessen in diesem Format gelesen werden:

let a = 1 in
  let b = 2 in
    a + b

Welches in Reason folgendes ergibt:

let a = 1;
let b = 2;
a + b;

Was dich erinnern könnte an:

/* Reason syntax */
(a) =>
  (b) =>
    a + b;

Obwohl sie nicht genau die gleichen sind, kannst du hoffentlich sehen, dass let lediglich ein Ausdruck ist und einer Funktion ähnelt! Um eine visuelle Vertrautheit zu erzeugen, haben wir in Reason in durch ; ersetzt; Aber lass dies nicht die zugrundeliegende Eleganz von Ausdrücken verbergen.
