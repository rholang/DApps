Installation

Reason est fourni par défaut dans BuckleScript, notre compilateur qui convertit le code Reason en code JavaScript.

Prérequis: soit npm (fourni avec node) ou Yarn.

Pour installer BuckleScript & Reason en global :

yarn global add bs-platform

(ou npm install -g bs-platform pour npm).

Nouveau projet

L'installation globale est fournie avec un simple générateur de projet. Essayer :

bsb -init my-new-project -theme basic-reason

Pour compiler et excuter le projet que vous venez de créer :

cd my-new-project
yarn build # or npm run build, for npm
node src/Demo.bs.js

C'est tout ! Nous utilisons BuckleScript pour compiler Reason en JavaScript, puis nous utilisons NodeJS pour exécuter le JavaScript. N'hésitez pas à utiliser les fichiers JS générés de la manière que vous souhaitez, comme si vous les aviez écrits vous-même.

Lors du développement, au lieu d'exécuter npm run build à chaque fois pour compiler, exécutez npm run start pour démarrer un watcher qui recompile automatiquement lorsque des fichiers sonnt modifiés.

Par défaut, le thème basic-reason configure BuckleScript pour qu'il renvoie les fichiers JS générés avec les fichiers Reason à partir desquels ils ont été produits. Voici la raison pour laquelle nous faisons cela. Si vous préférez conserver les fichiers générés ailleurs, modifiez bsconfig.json pour définir in-source sur false. Les fichiers JS seront générés dans le dossier lib/js à la place.

Sinon, pour démarrer une application ReasonReact, suivez les instructions ici.

Projet existant

Vous pouvez installer la toolchain en local dans un projet existant, à l'aide de la commande habituelle :

yarn add --dev bs-platform

(ou npm install --save-dev bs-platform pour npm).

Le reste est identique à ce qui est écrit plus haut.
