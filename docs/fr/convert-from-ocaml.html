Convertir depuis OCaml

La conversion d'un projet d'OCaml à Reason est simple et ne nécessite pas de changements sémantiques. Cependant, quelques configurations de build et changements divers sont requis.

Makefile

Si votre système de build exécute des commandes de build explicites, alors la façon la plus simple d'utiliser Reason avec ocamlopt/ocamlc est d'ajouter les flags suivant à chaque étape de compilation :

# intf-suffix dit au compilateur où regardez pour trouver les fichiers d'interface correspondants
ocamlopt -pp refmt -intf-suffix rei -impl myFile.re
ocamlopt -pp refmt -intf myFile.rei

ocamlbuild

Il n'y a pas de support officiel de syntaxe Reason pour ocamlbuild bien que vous puissiez écrire des règles personnalisées pour ocamlbuild qui utiliseront vos fichiers .re/.rei correctement. Ce diff supprime un utilitaire legacy qui peut être utilisé comme référence.

Dune

Si votre projet utilise Dune alors la syntaxe Reason est automatiquement supportée. Ajoutez une dépendance au paquet Reason, alors Dune prêtera attention aux fichiers .re/.rei dans votre projet.

# Si vous utilisez esy
esy add @esy-ocaml/reason

# Si vous utilisez opam
opam install reason

Convertir depuis OCaml

Le code OCaml peut être (principalement) automatiquement converti en Reason en utilisant refmt. Voir refmt --help pour plus d'options :

refmt --parse ml --print re yourFile.ml > yourFile.re
refmt --parse ml --print re yourFile.mli > yourFile.rei

Nettoyer l'output : Le code Raeason converti peut attacher [@implicit_arity] aux constructeurs de variants de la sorte : [@implicit_arity]C(1, 2). Cela est dû au fait qu'OCaml à cette syntaxe ambiguë où un constructeur à plusieurs arguments attend l'argument sous la forme d'un tuple. Il n'est donc pas clair si le code OCaml C (1, 2) doit être traduit en C (1, 2) ou C 1 2 en Reason. Par défaut, refmt le convertira en [@implicit_arity]C(1, 2), qui indique au compilateur ceci peut être soit C(1, 2) soit C((1, 2)). La balise "implicit arity" s'assure que l'output recrée la sémantique d'OCaml afin qu'il soit correctement compiler une fois converti en Reason.

Vous pouvez facilement nettoyer ce qui vous est renvoyeé par refmt en indiquant que les balises [@implicit_arity] soient enlevées. Passez --assume-explicit-arity à refmt pour forcer le formateur à générer C(1, 2) au lieu de [@implicit_arity]C(1,2). Cette approche générera une syntaxe Reason plus propre mais vous devrez peut-être en convertir sélectivement quelques variants C(1,2) en C((1,2)) pour que le type checker soit content.

Il existe déjà quelques règles d'exception internes pour couvrir les constructeurs communs qui nécessitent un seul tuple comme argument, donc refmt les convertiroa correctement sans avoir besoin de votre nettoyage manuel (par exemple, Some (1, 2) en OCaml sera converti en Some((1, 2)) au lieu de Some(1, 2)).

Pour fournir votre propre liste d'exception, dans votre projet qui attend un seul tuple comme argument, créez un fichier contenant tous les constructeurs (sans préfixe de module) séparés par des lignes, et utilisez --heuristics-file <filename> pour dire à refmt que tous les constructeurs listés dans le fichier seront traités comme constructeurs avec un seul tuple comme argument :

> cat heuristics.txt
  TupleConstructor
  And
  Or

> cat test.ml

type tm =
  TupleConstructor of (int * int)
| MultiArgumentsConstructor of int * int
let x = TupleConstructor(1, 2)
let y = MultiArgumentsConstructor(1, 2)
module Test = struct
  type a = | And of (int * int) | Or of (int * int)
end;;
let a = Test.And (1, 2)
let b = Test.Or (1, 2)
let c = Some (1, 2)

Ainsi, seuls les constructeurs qui ont été listés seront considéré comme acceptant des tuples à la place de plusieurs arguments.

> refmt --heuristics-file \
    ./heuristics.txt --assume-explicit-arity \
    --parse ml --print re test.ml

type tm =
  | TupleConstructor((int, int))
  | MultiArgumentsConstructor(int, int);

let x = TupleConstructor((1, 2));

let y = MultiArgumentsConstructor(1, 2);

module Test = {
  type a =
    | And((int, int))
    | Or((int, int));
};

let a = Test.And((1, 2));

let b = Test.Or((1, 2));
