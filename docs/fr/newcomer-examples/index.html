Exemple pour Débutants

Un exemple vaut mieux que mille mots.

Cette section est dédiée aux nouveaux venus essayant de comprendre les notions et conventions générales en Reason et BuckleScript. Si vous êtes un débutant qui a une bonne idée pour un exemple, n’hésitez pas à suggérer une modification !

Utilisation du type option

option est un variant qui vient avec la bibliothèque standard. Il évite le besoin de valeur nulles des autres langages.

let possiblyNullValue1 = None;
let possiblyNullValue2: option(string) = Some("Hello@");

switch (possiblyNullValue2) {
| None => print_endline("Nothing to see here.")
| Some(message) => print_endline(message)
};

Création d’un type paramétré

type universityStudent = {gpa: float};

type response('studentType) = {
  status: int,
  student: 'studentType
};

let result: response(universityStudent) = fetchDataFromServer();

Créer un objet JavaScript

En partant du principe que vous compilez vers JavaScript.

[@bs.deriving abstract]
type payload = {
  name: string,
  age: int
};

let obj1 = payload(~name="John", ~age=30);

/* Compile en un object JS avec les champs ci-dessus */

Notez que le résultat ci-dessus n'est pas un record; c'est une fonctionnalité BuckleScript spéciale.

Vérifier des types JS "nullables" en utilisant le type option

Pour une fonction qui reçoit une valeur JavaScript qui est potentiellement null ou undefined, il est idiomatique de le convertir en un option Reason. La conversion est faîte via une fonction helper dans le module Js.Nullable de Bucklescript. Dans ce cas, toOption:

let greetByName = (possiblyNullName) => {
  let optionName = Js.Nullable.toOption(possiblyNullName);
  switch (optionName) {
  | None => "Hi"
  | Some(name) => "Hello " ++ name
  }
};

Cette vérification compile en possiblyNullName == null en JS, et vérifie donc la présence de null ou undefined.
