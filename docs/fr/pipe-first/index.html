Pipe First

-> est un opérateur pratique qui permet de passer le résultat d'une expression en tant que premier paramètre à une autre expression. a(b) devient b->a. C'est une syntaxe qui n'a aucun coût d'exécution.

Imaginez que vous ayez :

validateAge(getAge(parseData(person)))

Le code est quelque peu difficile à lire, l'appel à une fonction peut être tellement imbriqué que la logique devient difficile à suivre. Utilisez l'opérateur pipe first pour simplifier cela

person
  ->parseData
  ->getAge
  ->validateAge

Fondamentalement, parseData(person) est transformé en person->parseData et getAge(person->parseData) est transformé en person->parseData->getAge, etc.

Cette technique fonctionne aussi si la fonction accepte plusieurs arguments.

a(one, two, three)

est identique à

one->a(two, three)

Cela fonctionne aussi avec des arguments labellisés.

Conseils & astuces

Essayez de ne pas abuser des pipes ; ils sont un moyen pour parvenir à une fin. Les nouveaux arrivants façonnent parfois l'API d'une bibliothèque pour profiter du pipe. C'est plutôt une régression.

Par convention, nous ne transformons pas l'appel de fonction le plus imbriqué en un pipe. Ainsi, l'exemple ci-dessus sera généralement écrit comme :

parseData(person)
  ->getAge
  ->validateAge

Chaînage de méthodes JS

Cette section nécessite une compréhension de l'API de binding Bucklescript.

Les API JavaScript sont souvent attachées à des objets et souvent chaînables, comme suit :

const result = [1, 2, 3].map(a => a + 1).filter(a => a % 2 === 0);

asyncRequest()
  .setWaitDuration(4000)
  .send();

En supposant que nous n’ayons pas besoin du chaînage ci-dessus, nous lierons (bind) this à chaque cas en utilisant bs.send de la section précédente :

[@bs.send] external map : (array('a), 'a => 'b) => array('b) = "map";
[@bs.send] external filter : (array('a), 'a => 'b) => array('b) = "filter";

type request;
external asyncRequest: unit => request = "asyncRequest";
[@bs.send] external setWaitDuration: (request, int) => request = "setWaitDuration";
[@bs.send] external send: request => unit = "send";

Vous les utiliseriez comme ceci :

let result = filter(map([|1, 2, 3|], a => a + 1), a => a mod 2 == 0);

send(setWaitDuration(asyncRequest(), 4000));

Ce qui a l'air d'être bien pire que la contrepartie JS ! Nous devons maintenant lire la logique dans le sens inverse. Nous pouvons aussi ne pas utiliser l'opérateur |> ici, étant donné que l’objet vient en premier dans le binding. Mais -> fonctionne !

let result = [|1, 2, 3|]
  ->map(a => a + 1)
  ->filter(a => a mod 2 === 0);

asyncRequest()->setWaitDuration(4000)->send;

Pipe dans les Variants

Les constructeurs de variants, comme Some ou Student, ressemblent à des fonctions, mais malheureusement n'en sont pas, dû à quelques raisons historiques. Parfois, il serait bien pratique de pouvoir s’en servir comme des fonctions. Pipe First saisit cette opporttunité pour vous permettre de le faire !

let result = name->preprocess->Some

Nous transformons ceci en :

let result = Some(preprocess(name))

Notez qu’utiliser un constructeur de variant comme une fonction ne fonctionnerait nulle part ailleurs.

Pipe Placeholders

Un placeholder s'écrit via un underscore et indique a Reason que vous voudrez fournir l'argument d’une fonction plus tard. Les deux déclaration qui suivent sont équivalentes :

let addTo7 = (x) => add3(3, x, 4);
let addTo7 = add3(3, _, 4);

Parfois, vous ne voulez pas donner la valeur que vous avez dans la première position. Dans ces cas, vous pouvez marquer une valeur de placeholder pour montrer sur quel argument vous souhaitez utiliser le pipe.

Disons que vous avez une fonction namePerson, qui prend une person puis un name en arguments. Si vous transformez une personne, alors le pipe fonctionnera comme il est :

makePerson(~age=47, ())
  ->namePerson("Jane");

Si vous avez un name que vous voulez appliquer à un objet person, vous pouvez utiliser un placeholder :

getName(input)
  ->namePerson(personDetails, _);

Cela vous permet de vous diriger vers n'importe quel argument de position. Il fonctionne également pour les arguments nommés :

getName(input)
  ->namePerson(~person=personDetails, ~name=_);
