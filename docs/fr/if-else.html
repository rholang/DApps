If-Else

Les ifs en Reason sont des expressions, ils évaluent leur contenu :

let message = if (isMorning) {
  "Good morning!"
} else {
  "Hello!"
};

Note : une expression if-elseif-else sans la branche else finale produit implicitement un (). Donc ceci :

if (showMenu) {
  displayMenu()
};

est fondamentalement identique à :

if (showMenu) {
  displayMenu()
} else {
  ()
};

Qui est généralement ce que vous attendez, displayMenu() doit retourner un unit aussi.

Voici une autre façon de le regarder. Ceci est clairement faux :

let result = if (showMenu) {
  1 + 2
};

Il donnerait un erreur de type, disant essentiellement que la branche implicite else à un type unit tandis que la branche if a un type int. Intuitivement, cela a du sens : quelle valeur de result serait-ce, si showMenu était false ?

Nous avons aussi le sucre syntaxique ternaire.

let message = isMorning ? "Good morning!" : "Hello!";

Utilisation

if-else et le ternaire sont beaucoup moins utilisés en Reason que dans d'autres langages. Le pattern-matching supprime toute une catégorie de code qui nécessitait auparavant des conditionnels. Ne préférez if-else que si vous n'avez, disons, que deux branches par exemple.

Décisions de conception

Le ternaire Reason est juste un sucre syntaxique pour la variant bool et un switch :

switch (isMorning) {
| true => "Good morning!"
| false => "Hello!"
}

Si vous passez ce bout de code dans refmt, vous aurez :

isMorning ? "Good morning!" : "Hello!";
