String & Char

String

Las cadenas en Reason se delimitan usando comillas dobles (las comillas simples están reservadas para el siguiente tipo de carácter).

let greeting = "Hello world!";
let multilineGreeting = "Hello
 world!";

Los caracteres especiales en la cadena necesitan ser escapados:

let oneSlash = "\\";

Para encadenar cadenas, usa ++:

let greetings = "Hello " ++ "world!";

Cadenas entrecomilladas

Hay una sintaxis especial para las cadenas que permite

  • cadenas multilínea igual que antes
  • ningún carácter especial para escapado
  • enlaces para pre procesadores especiales
let greetingAndOneSlash = {|Hello
World
\
Hehe...
|};

Hablando analógicamente, es como la interpolación de cadenas entre comillas invertidas de JavaScript, excepto sin necesidad de escapar los caracteres especiales y sin interpolación de variables integrada. Aunque puedes restaurar esta última funcionalidad de manera trivial, como ha hecho BuckleScript:

let world = {js|世界|js}; /* Soporta Caracteres Unicode */
let helloWorld = {j|你好，$world|j}; /* Soporta Unicode y interpolación variables */

El pre procesador especial de BuckleScript puede entonces buscar los marcadores js y j en la cadena y transformarlo en otra cosa.

Uso

Más operaciones sobre cadenas se pueden encontrar en la biblioteca estándar. For JS compilation, see the familiar Js.String API in the BuckleScript API docs. Puesto que una cadena en Reason se mapea a una cadena de JavaScript, usted puede mezclar y hacer coincidir con las operaciones de cadenas en ambas bibliotecas estándar.

Consejos y trucos

https://twitter.com/jusrin00/status/875238742621028355

¡ Ahora tienes un sistema de tipos expresivo! En un lenguaje sin tipos, a menudo se sobrecarga el significado de cadena utilizandolo como:

  • un id único: var BLUE_COLOR = "azul"
  • un identificador en una estructura de datos: var BLUE = "blue"; var RED = "red"; var colors = [BLUE, RED]
  • el nombre de un campo de un objeto: person["age"] = 24
  • un enum: if (audio.canPlayType() === 'probably') {...} (ಠ_ಠ)
  • pronto encontrarás horrible otros patrones disparatados, después de acostumbrarse a las alternativas de Reason.

Cuanto más sobrecargas el pobre tipo de cadena, ¡menos puede ayudarte el sistema de tipos (o un compañero)! Reason proporciona datos y tipos de estructuras alternativas concisos, rápidos y fácil de mantener para los casos de uso anteriores (por ejemplo, las variantes, en una sección posterior).

Bajo compilación nativa, las cadenas Reason compilan a una representación simple cuyo rendimiento es fácil de analizar, a expensas de que a veces requieren ajuste de rendimiento manual. Por ejemplo, ingenuamente encadenar cadenas como "hi " ++ "how " ++ "are " ++ "you?" innecesariamente asigna memoria para las cadenas intermedias "are you?" y "how are you?" (aunque se puede optimizar en una sola cadena en estos casos simples). En este caso, elige String.concat. En cierto modo, es algo sutil que el análisis en tiempo de ejecución tradicional que hemos aprendido en la escuela puede finalmente ser útil de nuevo.

Under JavaScript compilation, a Reason string maps to a JavaScript string and vice-versa, so no such above concern or analysis opportunities apply.

Decisiones de Diseño

La característica de no escapar caracteres especiales en las cadenas entrecomillada permite meticulosas DSLs como las expresiones regulares:

let r = Str.regexp({|hello \([A-Za-z]+\)|});

al contrario de

let r = Str.regexp("hello \\([A-Za-z]+\\)");

Aunque para la compilación JS, usarás [%bs.re] y Js.Re, ya que Str no está disponible.

Reason/OCaml enfatiza en la sencillez sobre lo ingenioso puede verse aquí a través de su sencilla implementación de cadena nativa. Una implementación de cadena demasiada sofisticada puede a veces ser contraproducente.

Caracteres

Reason tiene un tipo para una cadena de una sola letra:

let firstLetterOfAlphabet = 'a';

Nota: Char no soporta caracteres Unicode o UTF-8.

Consejos y trucos

Un carácter compila en un rango de entero entre 0 y 255, para velocidad extra. Puedes usar también el patrón de coincidencia (cubierto más adelante) en él:

let isVowel = (theChar) =>
  switch (theChar) {
  | 'a' | 'e' | 'i' | 'o' | 'u' | 'y' => true
  | _ => false
  };

Para convertir una cadena a un Char, usa "a".[0]. Para convertir un carácter en una cadena, utilice String.make(1, 'a').
