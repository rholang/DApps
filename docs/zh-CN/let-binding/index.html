Let Binding

“let 绑定” 在其他语言里，可能叫做 “变量申明”。let 给值赋予名字。他们可以被后面的代码看见和引用。

let greeting = "hello!";
let score = 10;
let newScore = 10 + score;

块范围

Let绑定可以通过”{}“限定作用范围

if (displayGreeting) {
  let message = "Enjoying the docs so far?";
  print_endline(message)
};
/* 这里看不见 `message`！ */

Let绑定是不可变的

“不可变” 意思就是，“不能改变”。 一旦绑定引用了某个值, 它就不能引用其他任何内容 (除非它显式包含可变值, 稍后讨论)。 但是, 你可以创建一个同名的新绑定, 用来 * 屏蔽 * 是上一个绑定；之后，绑定将引用新分配的值。

let message = "hello";
print_endline(message); /* 打印 "hello" */
let message = "bye";
print_endline(message); /* 打印 "bye" */

技巧 & 技巧

由于绑定是通过 {} 限定的, 因此您可以在它们周围创建一个匿名范围:

let message = {
  let part1 = "hello";
  let part2 = "world";
  part1 ++ " " ++ part2
};
/* 这里看不见 `part1` 和 `part2`！ */

这可防止在这些行之后滥用绑定。

设计决策

原因是Reason是由OCaml编写的。OCaml 语法中的 "let绑定" 看起来像这样:

let a = 1 in
let b = 2 in
a + b

可以转换为这种格式来阅读

let a = 1 in
  let b = 2 in
    a + b

等同Reason中如下写法：

let a = 1;
let b = 2;
a + b;

这可能提醒你:

/* Reason 句法 */
(a) =>
  (b) =>
    a + b;

虽然它们不是严格相同的, 但希望您可以看到 let 只是一个表达式, 类似于函数! 在Reason中，, 我们将 in 转换为 ; ；看起来更加熟悉易懂，同时又能体现表达式的优雅。
