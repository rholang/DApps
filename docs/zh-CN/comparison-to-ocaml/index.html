Comparison to OCaml

如果你是一个OCaml的新手或者是一个正在学习OCaml的人，那么这个入门指导适合你。 但是别忘了 reason-tools可以自由的在OCaml和Reason语法间转换

注释

OCamlReason
(* OCaml (*nest*) *)/* Reason /*nest*/ */

运算符重命名

Reason有OCaml所有的中缀运算符，但是有几种运算符他们在表达上有区别。 在Reason中，结构相等写作==，引用（实际）相当写作 ===。 在Reason中，要想表达不相等，只要在等号字符前加上 !字符 (!= 是结构上不相等, 而!== 是引用不相等).

相等OCamlReason
结构x = yx == y
引用x == yx === y
不等OCamlReason
结构x <> yx != y
引用x != yx !== y

本地作用域

Reason的词法作用域和OCaml是完全相等的，但是在语法上更加类似于开发者熟知的‘‘块级作用域’’ 在Reason中，局部作用域由{}创建，在其中可能由let绑定和声明变量，并用;进行分割。 所有块计算到最后一行, 最后一行的分号是可选的。 只有当通过 ; 将多个项链在一起时, 才需要 {} 大括号。

<td>
  <pre><code>{

let msg = "Hello"; print_string(msg); let msg2 = "Goodbye"; print_string(msg2) };

OCaml 
let _ =
  let msg = "Hello" in
  print_string msg;
  let msg2 = "Goodbye" in
  print_string msg2

Reason的 {}移除了OCaml语法上的许多痛点。

  • 双分号被完全删除。
  • begin/end 被完全删除
  • 臭名昭著的命令式解析 问题 不见了
  • 模块体和局部作用域相统一

局部作用域 Vs 模块体

在Reason中，{} 块级作用域s和模块体内可以存在任何东西 通常可以在两个上下文之间剪切/粘贴代码。 在 OCaml 中, 两种上下文的语法非常不同。 本地作用域需要尾随 in, 但模块体不存在, 并且必须将某些命令性语句分配给 _ 或 (), 否则使用双 ;;。

注释参数

在Reason和 OCaml 中, 参数通过类型批注 (与其他内容一样), 在追加 : typeAnnotation 后将它们包装在括号中。

(arg: argType) => returnValue;

(arg: argType) => (arg2: arg2Type) => returnValue;

(arg: argType, arg2: arg2Type) => returnValue;

Reason和 OCaml 允许注释返回类型, 当使用 "let"的超级语法糖 形式。

(* OCaml *)
let myFunc (a:int) (b:int) :int * int = (a, b)
let myFunc (a:int) (b:int) :int list = [1]
let myFunc (a:int) (b:int) :int -> int = fun x -> x + a + b

/* Reason */
let myFunc = (a: int, b: int) :(int, int) => (a, b);
let myFunc = (a: int, b: int) :list(int) => [1];
let myFunc = (a: int, b: int): (int => int) => (x) => x + a + b;

因为我们在Reason中对所有函数都使用 = >, 所以有一种情况是, 我们需要在一个本身是函数类型的返回类型周围添加额外的括号。

类型参数

OCaml

OCaml 的类型应用程序 (认为 "泛型") 以相反的顺序应用。

在 OCaml中, 会导致一些直观的后果。

let x: int list = [2]

type listOfListOfInts = int list list

(* Parsed as: *)
type listOfListOfInts = (int list) list

当类型构造接受多个参数时，事情变得更加奇怪了。 多个参数需要括号和分号分割类型参数，但是这些括号不代表元组。 括号/逗号形式必须在(int, string) tuple这样的构造类型示例中给出。

type ('a, 'b) tuple = 'a * 'b

type listOfTuplesOfStringAndInt = (string, int) tuple list

(* Which is parsed as: *)
type listOfTuplesOfStringAndInt = ((string, int) tuple) list

(* Which allows a list of (tuples of (string and int)) *)
let tuples: listOfTuplesOfStringAndInt = [("asdf", 3)]

Reason

总之, Reason将几乎所有的语法都统一为简单的 "函数应用程序" 样式, 这意味着类型参数遵循语法中其他地方所看到的相同的逗号分隔模式。 这将导致学习的语法模式更少。

例如, 可以将 list 作为接受类型并返回新类型的类型的 "函数"。

作为类型参数的元组

因为 OCaml 使用括号和逗号来表示类型构造函数的多个参数, 所以当类型构造函数的一个参数本身是一个元组时, 它会混淆。 在 OCaml 中, 很难记住一个类型构造函数接受多个参数和一个类型构造函数接受一个恰好是元组的参数的区别。

下面的示例显示了将 * 两个 * 类型参数传递给 pair 和一个恰好为元组的 * 单个 * 类型参数之间的区别。

OCamlReason
type intPair = (int, int) pairtype intPair = pair(int, int)
type pairList = (int * int) listtype pairList = list((int, int))
  • 在Reason中, 表示元组或元组类型的语法总是看起来像元数组。
  • 在Reason中, 表示记录或记录类型的语法始终看起来像记录。
  • 其他一切都使用函数应用程序的语法模式 (逗号分隔参数)。

变体

OCaml
  • OCaml 已经期望构造函数参数类型以元组形式指定, 因此当一个构造函数需要一个 * 碰巧 * 为元组类型的单个参数时, 就会混淆。
  • 更让人费解的是, 构造函数 * 实际上 * 不接受元组, 但语法看起来像元组。
  • 有时, 用于实例化具有多个参数的构造函数的语法与构造变量的语法重叠, 该变量使用恰好是一个元组的单个参数, 因此它看起来像你支持了元组，事实上并没有。
Reason
  • 变量构造函数类型应以逗号分隔列表的形式列出, 并使用括号来分组优先级 (与 ** "其他内容" ** 一样)。
  • 构造变量的实例 (如您所猜到的) 遵循函数应用程序样式 (逗号分隔列表)。
  • 元组 ** 总是 ** * 看起来 * 类似元组, 而任何看起来像元数组 * 的内容都是 * 元组。

你能支出OCaml中的错误吗？ 这是一个在OCaml程序中司空见惯的错误。 嵌套的match 函数必须包裹在括号中，否则Some会被外部的match函数解析。 看起来就像这样。

let res = match x with
  | A (x, y) -> match y with
    | None -> 0
    | Some i -> 10
    | B (x, y) -> 0

Reason强制在switch外使用{}解决这个问题。

各种改进

OCaml不强制在序列(a;b;c;d) 或者元组(x,y)外面加括号。 但是Reason总是用{}去包住序列或者let声明，而且Reason总是要求元组要有()，其他更直观的语法则不需要用括号去包裹。

Lambdas 作为作用域不需要额外的上下文

这是一个受欢迎的改进，因为OCaml中的类型错误发生时，用户将会看到一个返回带有中缀,的元组。

<th>
  Reason
</th>

<td>
  <pre><code>let myFuncs = {

myFun: (x) => x + 1, your: (a, b) => a + b };

OCaml 
let myFuncs = {
  myFun = (fun x -> x + 1);
  your = (fun a b -> a + b);
}

Lambda 作为匹配结果不再需要额外的括号

各种差异

as 优先级

Reason中, as 的优先级高于 |。这允许在模式匹配中为整个行创建 as 别名。

<th>
  Reason
</th>

<td>
  <pre><code>let ppp =

switch (MyThing(20)) { | MyThing(x) as ppp | YourThing(x) as ppp => ppp };

OCaml 
let ppp = match MyThing 20 with
  | (MyThing x as ppp)
  | (YourThing x as ppp) -> ppp;

可变记录字段更新

由于Reason中等式及其否定更加一致, 因此 = 运算符可用于可变字段更新。

OCamlReason
myRec.field <- "next"myRec.field = "next"

前缀运算符

Reason的 ! 用于布尔型 not。Deferencing 使用后缀 ^。

OCamlReason
let x = !(foo.bar)let x = foo.bar^;
let x = !(foo#bar)let x = foo#bar^;
let x = !(!foo.bar)let x = foo^.bar^;
let x = !(!foo#bar)let x = (foo^)#bar^;
let x = !(!(foo.bar))let x = foo.bar^ ^;
let x = !(!(foo#bar))let x = foo#bar^ ^;
let x = !!(foo.bar)let x = !!foo.bar;
let x = !!(foo#bar)let x = !!foo#bar;
let x = !~(foo.bar)let x = !~foo.bar;
let x = !~(foo#bar)let x = !~foo#bar;

注释转义

因为Reason使用C类型的注释，一些模糊的自定义前缀/中缀必须区分开来。 这些前缀/中缀操作符的规则类似于OCaml的语法，但是下面这些例外。

特别的，如果如果前缀或者中缀字符的第一个字符是型号或者斜线，那么必须在前面加上反斜线。 在AST解析时反斜线会被去掉。 当打印时这些，反斜线会再次被加进去。

OCamlReason
let (/*) a b = a + blet (/\*) = (a, b) => a + b;
let x = 12 /-* 23 /-* 12let x = 12 /-* 23 /-* 12;

let (!=*) q r = q + rlet ( !=* ) = (q, r) => q + r;

let (!=/*) q r = q + rlet ( !=/\* ) = (q, r) => q + r;
let res = q (!=/*) rlet res = q(( !=/\* ), r);
