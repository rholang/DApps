Converting from JS

序言

在你继续阅读前，请确保Reason是你的团队所需要的。 尽管我们为Reason和BuckleScript的流行度做担保，但是不要因此惹恼了你的同事，并让他们对此产生差评。 否则那将很难消弭于无形。

这篇文档只覆盖了可帮助我们快速高效转换代码的流程。 并不打算覆盖所有 语言/FFI 的功能（即便也有提及）。 假设您对 Reason/BuckleScript 已有一定的了解。

语法

目标：第一要务，使代码语法有效。 诸如类型错误、缺少模块、不合适的文件组织、太多 externals 等问题，都先放在一边。 回归测试确定 “没有语法错误” 后，会回头清理这些问题。

由于 Reason 语法同 JavaScript 相似，不必创建一个新的 Reason 文件，拷贝已有的 js 文件，在这上修改就好了。

提示：别忘了可以在编辑器/终端上使用 refmt！ 如果不知道操作符的优先级，将它们包在括号里就好，多加几个也无妨，refmt 代码后，再看看还剩下哪些。 同样，也不必在缩进和空格上浪费时间；refmt 会处理好它们。

/* 拷贝来的原 JavaScript 文件 */
const school = require('school');

const defaultId = 10;

function queryResult(usePayload, payload) {
  if (usePayload) {
    return payload.student
  }
  return school.getStudentById(defaultId);
}

这一步，你会需要做下面几件事：

  • 转换函数调用的语法。
  • 转换 var/const 至 let。
  • 隐藏 require。
  • 其他修改。BuckleScript 没有提供等价语法的部分，使用 bs.raw（文档）。

重申，只用操心让代码语法上合法。 如果在转换文件时，同时还想学习所有的语法、类型等语义，这会降低你的迭代速度。

/* 语法有效但语义错误  */
/* const school = require('school'); */
let defaultId = 10;

let queryResult = (usePayload, payload) =>
  if (usePayload) {
    payload.student
  } else {
    /* if-else 是个表达式，在 Reason 中不用提前 return */
    school.getStudentById(
      defaultId
    )
  };

类型，第 1 关

目标：修正类型，只用做到不影响下一步即可。

也许还会偶尔碰到些语法错误，但不会比上一步的更难搞。

  • 修改 foo.bar 成 foo##bar。 这个 BuckleScript 文档 会是你的好朋友。
  • 转换 {foo: bar} 至 [%bs.obj {foo: bar}]（docs）。 refmt 会把这改成 {"foo": bar}。
  • 与外部的 JS 文件协作，使用 external。这是 BuckleScript 的 外部函数接口.
    • 内联 externals。现在不需要简洁划分良好的 externals 文件。之后会解决。
    • 如果觉得正确标记 external 的类型过于麻烦，可以使用一些多态类型来占位，例如 external getStudentById: 'whatever => 'whateverElse = ...。
    • 对于那些难以恰当转换的数据类型和模式，有时也可以像这样，external unsafeCast : myPayloadType => anotherDataType = "%identity";.

这是第一关；最终的类型可能会看起来不一样。 该领奖品了！ 一旦修好了所有的类型错误，这时候， JS 文件就能自动生成。 并排打开两个文件。 是时候看看之前留下的问题了！

/* 语法合法，语义错误，但也好些了 */
[@bs.module "school"] external getStudentById : 'whatever => 'whateverElse = "getStudentById";

let defaultId = 10;

let queryResult = (usePayload, payload) =>
  if (usePayload) {
    payload##student /* this will be inferred as `Js.t 'a` */
  } else {
    getStudentById(defaultId)
  };

运行时语义

目标：修正生成的 JS 里的所有错误。

对比你之前的 JS 文件。结果可能是错的；可能是一些东西的用法错了，也可能是 externals 的类型错了。

  • 输入 JS 对象（那些用到了 ## 的部分）。
  • 转换成 records/variants/idiomatic 的 OCaml 类型。

在这过程中不断查看每次修改的输出结果。

type student; /* 抽象类型，后面会提到 */

[@bs.module "school"] external getStudentById : 'whatever => student = "getStudentById";

type payloadType = {. "student": student};

let defaultId = 10;

let queryResult = (usePayload, payload: payloadType) =>
  if (usePayload) {
    payload##student
  } else {
    getStudentById(defaultId)
  };

收工（类型，第 2 关）

目标：确保类型合法（sound）。

修复第一关留下的问题。

  • 确保 external 没有留下任何 'whatever 类型。
  • 可以保持 external 内联，或者放到另外的文件。
/* 当前文件 */
type payloadType = {. "student": School.student}; /* TODO: 放在别的地方！ */

let defaultId = 10;

let queryResult = (usePayload, payload: payloadType) =>
  if (usePayload) {
    payload##student
  } else {
    School.getStudentById(defaultId)
  };

/* School.re 文件 */
type student;

[@bs.module "School"] external getStudentById : int => student = "getStudentById";
[@bs.module "School"] external getAllStudents : unit => array(student) = "getAllStudents";

student 类型沒有实际內容；这个称为 抽象类型。 这是一种指定外部调用之间关系的便捷方式，无需知道数据的形状是什么。

这样就完成了！

提示

不要 试着一次就能完全转换所有 JS 文件到 Reason。 这种方式会拖慢你的速度！ 有 externals 和 bs.obj 留下也没问题，只不过暂时缺少了 OCaml 的一些功能（variants, 标签参数等）。 一旦转换好其他文件之后，藉由类型系统，可以回过头做 更快 的重构了。

无论你找到多棒的工具，把它们放在类似于 tempUtils.re 的文件里。这些是给你同事的简单范例，能省去一些沟通成本。

我们强烈建议 你把 JS 结果纳入版本控制。 它使你不用在构建系统集成上花费功夫，并确保当你不在场时，队友可以进行小的更改，审核输出的 diff 并捕获任何错误。 这也是个不错的卖点，纳入版本控制的 JS 结果对于紧急的 hot patches 也更友好（对 manager 来说也是一大卖点）。 而且升级 BuckleScript 版本的时候，也能发现输出代码的差异。 就像是 Jest snapshots，免费！

一如既往，在 Discord 或讨论区寻求帮助！
