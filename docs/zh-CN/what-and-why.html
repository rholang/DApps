What & Why

Reason 是什么？

Reason 并不是一门新语言，它是由一款久经沙场的语言 OCaml 驱动的新语法和工具链。 Reason 赋予了 OCaml 和 JavaScript 类似的语法，也迎合了前端程序员已经熟悉了的 NPM/YARN 工作流。

在这点上，Reason 几乎可以被认为是 JavaScript 的表兄弟了。它有牢固的静态类型，更快也更简洁，除去了 JavaScript 的历史包袱，加上了也许到 ES2030 才有的特性，还能同时使用 JS 和 OCaml 的生态！

感谢我们的合作项目 BuckleScript 让 Reason 可以被编译成 JavaScript，BuckleScript 可以将 OCaml/Reason 平滑编译成可读的 JavaScript 代码。 同时，感谢 OCaml 自身的能力，Reason 也可以编译成快速的准系统汇编。

为什么选择 Reason ？

“何苦学一门 全新 的语言呢？”

这不应该是不学习的原因。

事实上，OCaml 中 80% 的语义（又称如何运行）可以直接对应到现代的 JavaScript 上，反之亦然。你只需要很轻松的丢弃掉一点 JavaScript 边角语法，再学一些好的东西，就可以获得一门可以转换到非常可读 JavaSript 代码的语言，甚至是直接使用它 80% 的生态和工具。 这就是说，它可以被编译成准汇编，iOS，Android，甚至于微控制器！

我们还不清楚，究竟应该重塑 JavaScript 的哪些特性，才能符合一门快速语义和 100% 类型健壮的语言模型。 但是我们可以反过来看，从一门已经完善的语言中，考虑到性能和简单性，并对其进行一些调整，让它看起来更像是我们熟悉的 Web 语言中更好的一部分。

所有这些决定最终使得对于大部分常见的用例，Reason 的学习曲线并不比学习 JavaScript + 一个渐进的类型系统来得高。你会得到：

  • 坚如磐石的类型系统 OCaml 的类型具有 100% 的覆盖率（每一行代码），推导（类型可以被推导出来，无需手写），和健壮性（一旦编译完成，保证类型准确）。
  • 着眼于简洁和实用 为了熟悉度和交互操作，我们允许使用副作用，突变和对象，同时保证语言其他　部分的纯函数特性，不可变和函数式。
  • 关注性能和大小 Reason 的构建系统，bsb，可以在 100ms 内完成构建（增量式）。 产出的代码也很 精简。
  • 渐进式的学习和代码库迁移 从第一天开始就能感受到获得完整类型的好处。 如果其他方式失败了，可以直接在 Reason 的文件中粘贴上 JavaScript 的代码。
  • 强大的生态圈和工具链 使用你最爱的编辑器，你最爱的 NPM 包，以及任何你最爱的已有技术栈。

* 不相信我们？ 查看 JS -> Reason 备忘手册 或者在 playground 中尝试几个 Reason 的代码片段再看看右边的结果！

为什么选择 Ocaml 作为后端语言？而不是 [我最爱的那个语言]？

首先，请理解无论我们选择了哪门语言，你们中的绝大部分人都依然会问这个问题！=)

许多后端语言都满足上一节中提到的哪些要点。然而，以下的几点是我们最终没有选择其他语言的原因。

  • 编译到原生代码的能力 OCaml 的原生（汇编）启动时间在几毫秒内。 我们有计划让 Reason 运行在 native 上。同时，我们也关注通过 JavaScript 强大兼容性来实现的替代方案。
  • 副作用，可变操作和其他紧急方案 这些通常不是一门语言的闪光点。但是对于 Facebook 来说，能够在没有精心设计的互操作/重写的情况下拥有桥接代码库的能力是至关重要的。 OCaml 默认是不可变和函数式的代码，但是仍有让刚开始接触时可能会更简单的紧急方案。
  • 实现很重要 OCaml 经过了超过二十年的洗礼。 如果我们打算制造一个新的语法和工具链，我们希望它不会打破现有的语义，类型“陷阱和递减的受益，80% 在于编写代码库上。
  • 编写 React 的语言 Reason 的创造者 也创造了 ReactJS。ReactJS 的第一版原型也是使用 SML（OCaml 的远方亲戚）写的。 我们已经将 ReactJS 转换成了 ReactML 以供广泛使用。 几年后的现在，我们通过 ReasonReact 来迭代 ReactJS 的未来。
  • 友好，成长中的社区 我的意思是，我们非常友好。 我是加拿大人， 我们在世界各地都有成员。 如果找不到解决办法，可以再 Discord 频道中提问，至少有几个和你同时区的成员会为你解答。

不喜欢 Reason？

请先在 Discord 上和我们聊聊，防止有什么误会。 但是如果你依然不喜欢 Reason，这里有其他几个选择：

  • OCaml。Reason 只是构建在 OCaml 之上的语法和工具链层。OCaml<->Reason 之间的联通是无缝的，因为我们共享了一套抽象语法树。BuckleScript 在 Ocaml 中可以正常运行。 Js_of_ocaml 也可以。
  • Rust. 和我们很亲近的表兄弟！没有垃圾回收，专注于速度和安全。
  • Elm. 另一个表兄弟！可以制作简洁有趣的 webapp。
  • PureScript. 受 Haskell 启发，编译到 Web 端的语言。
  • Fable。基于 F#，与 OCaml 密切相关。
  • ClojureScript。表面上和我们没什么相似度，但同样优先考虑简洁和良好的互操作。
  • Swift。果味的 OCaml！
  • Haxe. 编译到几乎任何平台。
