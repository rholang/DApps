Syntax Cheatsheet

我们历经艰难让 Reason 在保留 OCaml 的强大语义和类型的同时看起来像 JS。希望你能喜欢！

Let 绑定

JavaScriptReason
const x = 5;let x = 5;
var x = y;无（感恩）
let x = 5; x = x + 1;let x = ref(5); x := x^ + 1;

String & Char

JavaScriptReason
"Hello World!"相同
'Hello World!'字符串必须用 "
Char 也是 String'a'
"hello " + "world""hello " ++ "world"

布尔值

JavaScriptReason
true, false相同
!true相同
&#124;&#124;, &&, <=, >=, <, >相同
a === b, a !== b相同
没有 deep equality （递归比较）a == b, a != b
a == b在对比中不存在隐式类型转换（感恩）

数字

JavaScriptReason
3相同 *
3.1415相同
3 + 4相同
3.0 + 4.53.0 +. 4.5
5 % 35 mod 3

* JS 里的整数和浮点数没有区别。

Object/Record

JavaScriptReason
没有静态类型type point = {x: int, mutable y: int}
{x: 30, y: 20}相同 *
point.x相同
point.y = 30;相同
{...point, x: 30}相同

* 这是 Reason 形式上等价，但不意味着会被编译成 JS 对象！想要编译到 JS 对象，看 这里。

数组

JavaScriptReason
[1, 2, 3][&#124;1, 2, 3&#124;]
myArray[1] = 10相同
[1, "Bob", true] *(1, "Bob", true)
没有不可变列表[1, 2, 3]

* JavaScript 可以使用 array 模拟 tuple，因为 JavaScript 的数组可以同时包含多种类型。

Null

JavaScriptReason
null, undefinedNone *

* 同样，只是形式上对应；Reason 没有 null 也没有关于 null 的 bug！ 但是当你需要 null 的时候，这有 一个 option 类型。

arg => retVal(arg) => retVal

add(4, add(5, 6))相同

柯里

JavaScriptReason
let add = a => b => a + blet add = (a, b) => a + b

JavaScript 和 Reason 都支持柯里，但 Reason 是 内置且针对避免分配 & 调用中间函数优化，只要有可能。

a ? b : c相同
switchswitch 以及 超强 能力。

* Reason 的条件语句总是一个表达式！

解构

JavaScriptReason
const {a, b} = datalet {a, b} = data
const [a, b] = datalet [&#124;a, b&#124;] = data *
const {a: aa, b: bb} = datalet {a: aa, b: bb} = data

* 好的编译器会提示也许 data 长度不是 2。使用 pattern-matching 替代。
