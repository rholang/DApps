Newcomer Examples

一個例子勝過千言萬語。

此章節獻給 Reason 及 BuckleSciprt 的新人，試圖找尋普遍風格 & 慣例。 如果你對範例有更好的點子，請提出編輯建議！

使用 option 型別

option 是個變體（variant）內建於標準函式庫。排除像是其他語言對於 null 值的需求。

let possiblyNullValue1 = None;
let possiblyNullValue2: option(string) = Some("哈嘍@");

switch (possiblyNullValue2) {
| None => print_endline("沒有東西")
| Some(message) => print_endline(message)
};

新增參數化型別

type universityStudent = {gpa: float};

type response('studentType) = {
  status: int,
  student: 'studentType
};

let result: response(universityStudent) = fetchDataFromServer();

新增 JS 物件

Assuming you're compiling to JavaScript.

[@bs.deriving abstract]
type payload = {
  name: string,
  age: int
};

let obj1 = payload(~name="John", ~age=30);

/* 編譯成 JS 物件，含有上述欄位 */

注意上述並不是 record，這是個特別的 BuckleScript 功能。

使用 option 型別檢查 JS nullable 型別

當函式的某些參數帶入的 JavaScript 值可能是 null 或者 undefined 時，Reason 語言習慣將它轉換成 option。 透過 BuckleScript 的 Js.Nullable 模組輔助函式進行轉換。 此範例，toOption：

let greetByName = (possiblyNullName) => {
  let optionName = Js.Nullable.toOption(possiblyNullName);
  switch (optionName) {
  | None => "Hi"
  | Some(name) => "Hello " ++ name
  }
};

這在 JS 將編譯成 possiblyNullName == null，用來檢查 null 或 undefined 是否存在。
