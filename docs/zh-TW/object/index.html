Object

Reason 裡大多數情況下，使用 record 將具名值放在一起。 某些適當情況下，不管用什麼方法，你可能希望使用相似功能，稱之為物件；比較彈性伴隨著不同的權衡。

如果你來自 JavaScript，你有很可能不是在找 Reason 物件，請直接跳到下方提示 & 技巧章節。

用法

型別宣告

物件不需要宣告型別， 雖然可以有一個。看起來像是 record，除了 .：

type tesla = {
  .
  color: string
};

開頭的點指出這是個"閉鎖"的物件型別，表示任何基於此型別的物件都必須完全符合其外型。

type car('a) = {
  ..
  color: string
} as 'a;

兩個點，又稱為省略，指出這是個"開放"物件型別，因此可以包含其他值及方法。 開放物件也是多形（多形）所以需要一個參數。

新增

Simple

type tesla = {
  .
  color: string,
};

let obj: tesla = {
  val red = "Red";
  pub color = red;
};

Js.log(obj#color) /* "Red" */

在這裡, 我們有一個簡單的物件包含 color方法和red屬性 。 此方法不帶參數, 並傳回私有屬性 red。 因為color method是一個public method，我們可以使用object notation物件標記法的方式訪問它。 請記住，物件只導出方法，所有屬性都是私有的。

Advanced

type tesla = {.
  drive: int => int
};

let obj: tesla = {
  val hasEnvy = ref(false);
  pub drive = (speed) => {
    this#enableEnvy(true);
    speed
  };
  pri enableEnvy = (envy) => hasEnvy := envy
};

物件型別是 tesla的物件有個公開方法 drive, 另外也包含私有方法 enableEnvy，私有方法只能在物件內部存取使用。

如你所見，Reason 物件也可以存取 this。JavaScript 物件的 this 行為可以反覆無常；但是Reason 的 this 永遠指向物件本身。

下列範例示範使用型別作為參數的開放物件型別。物件型別參數要求實作開放物件的所有方法。

type tesla('a) = {
  ..
  drive: int => int
} as 'a;

let obj: tesla({. drive: int => int, doYouWant: unit => bool}) = {
  val hasEnvy = ref(false);
  pub drive = (speed) => {
    this#enableEnvy(true);
    speed
  };
  pub doYouWant = () => hasEnvy^;
  pri enableEnvy = (envy) => hasEnvy := envy
};

你可以使用物件如下：

obj#doYouWant();

Tip & Tricks

如果你來自於 JavaScript，Reason 物件不會編譯成 JS 物件。 若有需要，請使用 BuckleScript 的特殊功能。
