Syntax Cheatsheet

我們致力讓 Reason 看起來像似 JS 並保有 OCaml 優良語意 & 型別。希望你喜歡！

JavaScriptReason
const x = 5;let x = 5;
var x = y;沒有（謝天謝地）
let x = 5; x = x + 1;let x = ref(5); x := x^ + 1;

JavaScriptReason

'Hello world!'字串必須始用 "

JavaScriptReason

物件/Record

JavaScriptReason
沒有靜態型別type point = {x: int, mutable y: int}
{x: 30, y: 20}一樣 *
point.x一樣
point.y = 30;一樣
{...point, x: 30}一樣

* 這是 Reason 的精神上等價；並不表示編譯成 JS 物件！若要編譯成後者，請見這裡。

陣列

JavaScriptReason
[1, 2, 3][&#124;1, 2, 3&#124;]
myArray[1] = 10一樣
[1, "Bob", true] *(1, "Bob", true)
沒有不可變 list[1, 2, 3]

* JavaScript 的陣列可以模擬 tuples，因為 JavaScript 陣列可以包含多種型別的元素。

Null

JavaScriptReason
null、undefinedNone *

* 同樣，只有精神上等價；Reason 沒有 nulls，也沒有 null 臭蟲！ 當你需要 null 特性時，可以用 option 型別。

Currying

JavaScriptReason
let add = a => b => a + blet add = (a, b) => a + b

JavaScript 和 Reason 都支援 currying，不過 Reason 的 currying 是內建且最佳化過，避免中介函式的配置 & 呼叫，無論在那。

switchswitch 但非常強大！

* Reason 條件式永遠是表達式！

JavaScriptReason

  const x = 23;
  const y = 34;
  return x + y;
})();
