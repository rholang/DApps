Tuple

Tuples 是

  • 不可變（immutable）
  • 順序的
  • 新增時便固定長度
  • 異質（heterogeneous）（允許包含不同型別的值）
let ageAndName = (24, "Lil' Reason");
let my3dCoordinates = (20.0, 30.5, 100.0);

Tuple 型別也可以用在型別標註。Tuple 型別看似 tuple 值。

let ageAndName: (int, string) = (24, "Lil' Reason");
/* tuple 型別別名 */
type coord3d = (float, float, float);
let my3dCoordinates: coord3d = (20.0, 30.5, 100.0);

注意：tuple 長度不會是 1。若長度為 1 就是值本身。

用法

標準函式庫提供 fst 以及 snd（這裡，位於 "Pair operations"）。這兩個方便的函式讓你可以直接存取 2-tuple 的第一及第二元素。 通常，存取 n-tuple 會使用解構（稍後在側邊欄會說明）：

let (_, y, _) = my3dCoordinates; /* 現在，你取得了 y */

_ 表示忽略 tuple 的這個成員。

Tuple 意味著不可變動；需要透過解構舊的來產生新的。

提示 & 技巧

若要傳遞多個值的情況，使用 tuple 十分方便。例如，傳回多個值：

let getCenterCoordinates = () => {
  let x = doSomeOperationsHere();
  let y = doSomeMoreOperationsHere();
  (x, y)
};

或者用在 "樣式匹配"（稍後會介紹）匹配各種可能：

switch (isWindowOpen, isDoorOpen) { /* 這是個 2-tuple */
| (true, true) => ...
| (true, false) => ...
| (false, true) => ...
| (false, false) => ...
}

盡量區域性的使用 tuple。長時間存在的資料結構，經常互相傳遞，偏好record，因為有欄位名稱。

Tuple 型別也會稱之為 "乘積型別（product type）"，且 (string, int) 在某些地方會寫成 string * int。 概念上 tuple 像是個 "笛卡爾積（cartesian product）"，想像一個 2D 網格，x 軸是 string y 軸是 int！

Tuple + switch 的結合十分強大且簡潔，且屏除整個 category 臭蟲。 結合在一起，清楚列出所有可能的組合值。 Tuple 型別 (bool, bool) 確實存在 2 * 2 = 4 種可能，型別系統要求你滿足 4 種可能。 這對程式碼重構十分良好；不再非正式地在這裡及那裡增加幾個 if-else，取而代之的是直接精確的 switch 分支，不會多也不會少。

設計決策

來自非型別語言的人可能對 tuple 的存在感到怪異。"為什麼不直接用 list/array？"

型別系統並不是萬能的，也不應該是；為了保持語言單純、效能（無論編譯或執行速度）及容易理解，需要某些品味上的取捨。 Reason lists，舉例來說，長度上更加彈性；可以被串聯、附加、切段等。 代價是，需要同質（一個清單中，只能有一種型別），隨機索引存取並非永遠有效*。 Tuple，換句話說，透過其限制長度使其更加快速，給予型別系統餘地詳盡地追蹤所有項目型別，並保證存取安全。 一般來說，你會注意到幾個型別系統突出、品味的取捨：record 欄位是固定的但可以異質，然而 map 的欄位彈性但同質等。

Reason 的 tuple 是"結構地"型別 。 表示不需要標註資料的明確型別，透過其用法及內容等等，編譯器也能推論出型別。 只要宣告以及用法推斷出來的外型相同，那就好了！

* 並非 Reason 型別系統不接受異質、動態長度的 lists；這是可以的（提示：GADT）！ 達到如此功能，增加了學習曲線及程式碼理解力。 就因為型別可以達成不代表讓某些片段程式碼無限制成長複雜是個好主意！
