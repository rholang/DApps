Comparison to OCaml

如果你來自 OCaml 或是正在閱讀 OCaml 所撰寫的教學，本指南是為你所準備！ 但請別忘了 reason-tools 可以直接進行 OCaml 與 Reason 句法轉換。

註解

OCamlReason
(* OCaml (*nest*) *)/* Reason /*nest*/ */

重新命名運算子（Operator）

Reason 包含所有 OCaml 的中綴運算子（infix operators），但某些運算子表達方式不同。 在 Reason，結構相等為 ==，而參考（實際）相等為 ===。 在 Reason，若要達到相對應的不相等，只需要將第 1 個字元改成 ! 即可。 （!= 結構不相等， !== 參考不相等）。

相等OCamlReason
結構x = yx == y
參考x == yx === y
不相等OCamlReason
結構x <> yx != y
參考x != yx !== y

區域範疇

Reason 的詞法範疇與 OCaml 一樣，但是 let 繫結的句法類似 "區域範疇"，大多數開發者所熟悉。 在 Reason，範疇使用括號 {} 建立，範疇內可以包含 let 繫結及命令的指令，使用 ; 分隔。 所有區塊都在最後一行評估，最後一行的分號可省略。 當你需串連多個 ; 時候才需要 {} 括號。

  Reason
</th>

<td>
  <pre><code>{

let msg = "Hello"; print_string(msg); let msg2 = "Goodbye"; print_string(msg2) };

OCaml 
let _ =
  let msg = "Hello" in
  print_string msg;
  let msg2 = "Goodbye" in
  print_string msg2

Reason 的 {} 句法移除了多數主要在 OCaml 句法的痛點：

  • 完全刪除雙分號。
  • 完全刪除 begin/end。
  • 不存在臭名昭彰的命令剖析問題。
  • 模組內部與區域範疇一致。

區域範疇 Vs. 模組內部

在 Reason，所有東西都可以用 {} 包在 區域範疇及模組內部。 通常可以在兩個語境（contexts）之間剪下貼上程式碼。 在 OCaml，兩個語境之間的句法差異很大。 區域範疇需用 in 結尾，模組內部則不需要，命令敘述必須指派 _ 或 () 又或者使用雙分號 ;;。

標註參數

在 Reason 與 OCaml，引數的型別標註（以及其他）在括號內的最後加上 :typeAnnotation。

(arg: argType) => returnValue;

(arg: argType) => (arg2: arg2Type) => returnValue;

(arg: argType, arg2: arg2Type) => returnValue;

當使用 "超填 let 繫結" 形式，Reason 與 OCaml 允許標註傳回型別。

(* OCaml *)
let myFunc (a:int) (b:int) :int * int = (a, b)
let myFunc (a:int) (b:int) :int list = [1]
let myFunc (a:int) (b:int) :int -> int = fun x -> x + a + b

/* Reason */
let myFunc = (a: int, b: int) :(int, int) => (a, b);
let myFunc = (a: int, b: int) :list(int) => [1];
let myFunc = (a: int, b: int): (int => int) => (x) => x + a + b;

因為 Reason 到處使用 =>，只有一種情境傳回型別需要額外括號，就是傳回型別本身是函式型別。

型別引數

OCaml

OCaml 的型別引數（想做是 “泛型”），係由反轉順序套用。

在 OCaml，存在某些不直覺的後果。

let x: int list = [2]

type listOfListOfInts = int list list

(* 剖析成： *)
type listOfListOfInts = (int list) list

當建構子接受多個引數時，更加奇怪。 多個引數必須使用括號及逗點分隔，這括號不代表 tuples。 當建構型別實例時，例如 (int, string) tuple，也必須使用括號/逗點形式。

type ('a, 'b) tuple = 'a * 'b

type listOfTuplesOfStringAndInt = (string, int) tuple list

(* Which is parsed as: *)
type listOfTuplesOfStringAndInt = ((string, int) tuple) list

(* Which allows a list of (tuples of (string and int)) *)
let tuples: listOfTuplesOfStringAndInt = [("asdf", 3)]

Reason

總之，Reason 幾乎統一所有語法成簡單的"函式應用"風格，表示型別參數也遵循相同逗點分隔的樣式。 結果是可以少學習一些句法樣式。

例如，想像 list 當作"函式"，接受一個型別傳回新的型別。 type and returns a new type.

OCaml

Tuples 作為型別參數

因為 OCaml 使用括號及逗點來表示型別建構子的多個參數，當參數本身是 tuple 時容易混淆。 OCaml 中，很難記住接受多個參數跟單個 tuple 參數的差別。

下列範例展示傳遞兩個型別參數到 pair 和單一必須是 tuple 的型別參數。

OCamlReason
type intPair = (int, int) pairtype intPair = pair(int, int)
type pairList = (int * int) listtype pairList = list((int, int))
  • 在 Reason，代表 tuples 或 tuple 型別的語法看起來都像 tuples。
  • 在 Reason，代表 records 或 record 型別的語法看起來都像 records。
  • 其他所有都是應用程式的句法樣式（逗點分隔參數）。

變異

OCaml
  • OCaml 已經預期建構子參數是 tuple 形式，當單一建構子預期單一參數且參數為 tuple 型別容易混淆。
  • 更加混淆的是，建構子實際上不接受 tuples 時，語法看起來像是 tuples。
  • 有時候初始多個參數的建構子的語法跟建構單一參數，參數為 tuple 的建構子語法重疊 - 當你實際上不是使用 tuple 時，看起來完全像是在使用 tuple。
Reason
  • 變異建構子型別預期用逗點分隔列出，使用括號來分組優先順序（其他也是一樣）。
  • 建構變異的實體（如你所猜想）遵循函式應用風格（逗點分隔列出）。
  • Tuples 永遠看起來像是 tuples，其他看起來像是 tuple 的 就是 tuple。

樣式匹配

<th>
  Reason
</th>

<td>
  <pre><code>let res = switch (x) {

| A((x, y)) => switch (y) { | None => 0 | Some(i) => 10 } | B((x, y)) => 0 };

OCaml 
let res = match x with
  | A (x, y) -> match y with
    | None -> 0
    | Some i -> 10
  | B (x, y) -> 0

你能發現 OCaml 範例的錯誤嗎？ 這是 OCaml 開發者中最常見的錯誤之一。 巢狀 match 必須包裝在括號裡面，否則某些情況會當作隸屬於外圍 match。 視覺上，實際是：

let res = match x with
  | A (x, y) -> match y with
    | None -> 0
    | Some i -> 10
    | B (x, y) -> 0

Reason 強制在 switch 使用 {} 來避免這個問題。

各種改進

OCaml 並不要求連續 (a;b;c;d) 或 tuples (x,y) 使用括號，所以終結其他非常方便語法規則。 由於 Reason 使用 {} 圍起連續或 let 繫結，且 Reason 要求 () 圍著 tuples，其他許多語法建構更加直覺表達，不需要額外括號。

Record 的 Lambdas 欄位不再需要額外括號

這是個受歡迎的改善，因為當函式傳回值是個 tuple 時，需要中置逗點 ,，使用者看到的 OCaml 型別錯誤時很容易混淆。

<th>
  Reason
</th>

<td>
  <pre><code>let myFuncs = {

myFun: (x) => x + 1, your: (a, b) => a + b };

OCaml 
let myFuncs = {
  myFun = (fun x -> x + 1);
  your = (fun a b -> a + b);
}

各種差異

as 優先順序

Reason 中，as 比 | 有更高優先順序。可以新增 as 別名供樣式匹配整個列使用。

  Reason
</th>

<td>
  <pre><code>let ppp =

可變的 Record 欄位改變

因為在 Reason 相等與否定已經更加一致，可變欄位的修改可以使用 = 操作子。

OCamlReason
myRec.field <- "next"myRec.field = "next"

前置運算子

Reason 的 ! 用做布林的 not。取值使用後置 ^。

OCamlReason
let x = !(foo.bar)let x = foo.bar^;
let x = !(foo#bar)let x = foo#bar^;
let x = !(!foo.bar)let x = foo^.bar^;
let x = !(!foo#bar)let x = (foo^)#bar^;
let x = !(!(foo.bar))let x = foo.bar^ ^;
let x = !(!(foo#bar))let x = foo#bar^ ^;
let x = !!(foo.bar)let x = !!foo.bar;
let x = !!(foo#bar)let x = !!foo#bar;
let x = !~(foo.bar)let x = !~foo.bar;
let x = !~(foo#bar)let x = !~foo#bar;

註解轉義

因為 Reason 使用 C-風格註解，某些不清楚的自訂前置/中置操作子必須用不同的方式撰寫。 前置/中置操作子的規則跟 OCaml 語法一樣，但有下列例外情況：

具體來說，前置/中置操作子的第一個字元是星號或正斜杠，必須使用反斜線跳脫。 當加入到 AST 會剖析除了反斜線的部份。 當重新印出時，跳脫反斜線將會自動加回。

OCamlReason
let (/*) a b = a + blet (/\*) = (a, b) => a + b;
let x = 12 /-* 23 /-* 12let x = 12 /-* 23 /-* 12;
let y = (/*) a blet y = a /\* b;
let (!=*) q r = q + rlet ( !=* ) = (q, r) => q + r;
let res = q (!=*) rlet res = q(( !=* ), r);
let (!=/*) q r = q + rlet ( !=/\* ) = (q, r) => q + r;
let res = q (!=/*) rlet res = q(( !=/\* ), r);
